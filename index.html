Welcome to Restpark

Minimal RESTful API for querying RDF triples

SPARQL is quite complex, and to implement it on top of existing storage solutions you need to use a proper triple store (RDF database), and be sure it implements all of SPARQL features correctly.

Wouldn't it be interesting to have a lightweight alternative to SPARQL which developers could implement themselves on top of their existing storage solution (MySQL, MongoDB, etc.) using the language they please (PHP, Java, Node.js)?

This is what Restpark is. Essentially a set of HTTP GET parameters that you need to handle, and respond with JSON.

Query parameters

  • ?subject to match a subject
  • ?predicate to match a predicate, and
  • ?object to match an object
  • ?limit to limit the result
  • ?offset to offset the result

Therefore a Restpark compatible query would look like
/restpark?subject={subject}&predicate={predicate}&object={object}

Why?

How do triple (RDF) datasets currently get published on the web? They're either available as large static .rdf files, which need to be downloaded and processed locally, or they exist behind SPARQL endpoints. By providing this RESTful API of your triple data, you allow users to access and query your data through a minimal interface.

Just imagine if all of RDF resources were published this way, you could quickly build applications that depend on these datasets, without having to download and process this information.

Welcome to Restpark

Minimal RESTful API for querying RDF triples

How do triple (RDF) datasets currently get published on the Web? They're either available as large .rdf files, which need to be downloaded and processed locally, or they exist behind complex SPARQL endpoints. By providing a RESTful API of your triple data, you allow users to access and query it through a simple interface based on just a couple of HTTP parameters.

Just imagine if RDF resources were published this way: you could quickly build applications that depend on these datasets, without having to download and process them locally.

This is what Restpark is. Essentially a set of HTTP GET parameters that you need to handle, and respond with JSON (be nice and provide also a JSONP callback).

Why?

SPARQL is quite complex, and to implement it on top of existing storage solutions you need to use a proper triple store (RDF database such as Jena), and be sure it implements all of SPARQL features correctly.

Having a lightweight alternative to SPARQL which developers could implement themselves on top of their existing storage solution (MySQL, MongoDB, etc.) using the language they want (PHP, Java, Node.js), would lower the entry barrier for Semantic Web data understanding and retrieval.

Query parameters

ParameterValueDescription
subject URI of subject Used to match a subject
predicate URI of predicate Used to match a predicate
object URI of object or literal Used to match an object such as a string
limit number Limit the number of triples in the result
offset number Skip n number of triples

Therefore a Restpark compatible query would look like
/restpark?subject={subject}&predicate={predicate}&object={object}
